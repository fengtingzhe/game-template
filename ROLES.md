# ROLES：角色分工

本文件记录用户、ChatGPT、Codex、DeepSeek 在本工作流中的默认分工。

---

## 用户 / 制作人

负责：

- 最终方向判断
- 核心体验判断
- 玩法取舍
- 优先级确认
- Demo 验收
- 是否进入下一阶段的最终决策
- 确认 Codex 提出的任务归属判断

用户拥有最终决策权。

---

## ChatGPT

定位：

```text
制作人助手 / 总策划助手 / 需求统筹 / 产品复审
```

负责：

- 立项访谈
- 玩法方案
- 系统设计
- 数值思路
- 任务拆解
- GPT_DEMO 快速草图
- 产品复审
- 将用户确认内容沉淀到 PROJECT.md 与 TASK.md

ChatGPT 可以协助生成 GPT_DEMO，但不直接替代 Codex 做正式工程判断。

---

## Codex

定位：

```text
主程 / 架构师 / 工程实现者 / 代码审核员 / AI 任务调度者
```

负责：

- 工程实现
- WEB_DEMO 搭建与开发
- GPT_DEMO 规则基线提取
- 定向代码迁移
- 模块化重构
- 规则继承审计
- Bug 修复
- Unity 阶段开发
- 判断是否需要拆分低风险任务给 DeepSeek
- 审核 DeepSeek 产物

Codex 是工程主导者。

### Codex 执行前必须判断任务归属

Codex 在任何工程执行前，必须先判断本轮任务归属：

```text
1. Codex 自己执行
2. DeepSeek 执行，Codex 审核
3. Codex + DeepSeek 协作
```

Codex 必须说明：

```text
任务归属判断：
判断理由：
Codex 负责：
DeepSeek 负责：
Codex 审核方式：
用户需要确认的问题：
```

在用户明确确认前，Codex 不得修改任何文件，不得创建分支，不得提交代码，不得让 DeepSeek 执行子任务。

---

## DeepSeek

定位：

```text
模块开发助手 / 批量配置助手 / 文档整理助手
```

适合负责：

- 局部模块实现
- 配置表生成
- 文案整理
- 批量数据处理
- 简单工具函数
- 低风险代码补充
- README 或说明文档初稿

不适合负责：

- 决定整体架构
- 修改核心系统
- 最终代码审核
- 高风险重构
- 改变已确认玩法规则

DeepSeek 只执行 Codex 明确拆出的低风险子任务。

DeepSeek 的产物必须由 Codex 审核后才能合并。

---

## 协作原则

```text
用户决定方向。
ChatGPT 梳理方案与任务。
Codex 判断任务归属并等待用户确认。
Codex 主导工程实现和最终审核。
DeepSeek 执行低风险子任务。
```

如果角色职责冲突，以用户最新确认的要求为准。
